# Episode 23: Splitting Auth Service

## 🎯 Objectives
* Create a new microservice folder called `auth-service` inside the `microservices` folder in our monorepo.
* Register the new service to the `go.work` workspace.
* Isolate the **Authentication** domain (login, token generation, refresh token rotation, and blacklisting) to run as an independent service on port `:8081`.

---

## 📐 Auth Service Splitting Concept
In our microservices architecture, **Auth Service** handles session management and token security:
1. **Clean Architecture with Repository Pattern:** Database query logic has been cleanly separated from the service layer by introducing `UserRepository` and `RefreshTokenRepository`.
2. **Shared Modules (Clean Architecture):** Generic helper functionalities (database connections, logging, configuration loading, JWT utilities, and error handling) are imported from the `shared` module rather than duplicated inside the auth service.
3. **Efficiency:** By separating Auth Service, if login traffic increases sharply, we can scale up only the Auth Service container without duplicating other resources.

## 📦 Steps

### Step 1: Create auth-service Folder
Create folder structure for `auth-service` (under `microservices/`):
```bash
mkdir -p auth-service/cmd auth-service/internal/auth/{handler,service,repository,model}
```

Initialize Go Module inside `auth-service` folder:
```bash
cd auth-service
go mod init github.com/bashocode/gowallet/microservices/auth-service
cd ..
```

Register this new service to Go Workspace in `go.work` file:
```bash
go work use ./auth-service
```

### Step 2: Reference Shared Packages
Instead of copying files from the monolith, import shared helpers directly from the **`shared`** package:
* Logger: `github.com/bashocode/gowallet/microservices/shared/logger`
* Database Connection: `github.com/bashocode/gowallet/microservices/shared/database`
* Error Middleware: `github.com/bashocode/gowallet/microservices/shared/middleware`
* Config Loader: `github.com/bashocode/gowallet/microservices/shared/config`
* JWT: `github.com/bashocode/gowallet/microservices/shared/auth`

### Step 3: Setup Entry Point `auth-service/cmd/main.go`
Create `main.go` file in `auth-service/cmd/main.go` to run Auth Service HTTP server on port `:8081`:

```go
package main

import (
	"log"

	"github.com/bashocode/gowallet/microservices/auth-service/internal/auth/handler"
	"github.com/bashocode/gowallet/microservices/auth-service/internal/auth/repository"
	"github.com/bashocode/gowallet/microservices/auth-service/internal/auth/service"
	"github.com/bashocode/gowallet/microservices/shared/config"
	"github.com/bashocode/gowallet/microservices/shared/database"
	"github.com/bashocode/gowallet/microservices/shared/logger"
	"github.com/bashocode/gowallet/microservices/shared/middleware"
	"github.com/gin-gonic/gin"
)

func main() {
	// Logger initializes automatically on import, but InitLogger remains available
	logger.InitLogger()
	logger.Log.Info("Starting Auth Microservice...")

	cfg := config.LoadConfig()

	// Connect to Redis (for token blacklisting)
	rdb, err := database.ConnectRedis(cfg.RedisAddr)
	if err != nil {
		log.Fatalf("Could not connect to Redis: %v", err)
	}
	defer rdb.Close()

	// Connect to MySQL (for refresh tokens & user checks)
	db, err := database.ConnectWithRetry(cfg.DBDSN)
	if err != nil {
		log.Fatalf("Could not connect to database: %v", err)
	}
	defer db.Close()

	// Initialize layers
	rtRepo := repository.NewMySQLRefreshTokenRepository(db)
	userRepo := repository.NewMySQLUserRepository(db)
	authSvc := service.NewAuthService(rdb, rtRepo, userRepo)
	authHandler := handler.NewAuthHandler(authSvc)

	r := gin.New()
	r.Use(gin.Recovery())
	r.Use(middleware.ErrorHandler())

	// Auth Routes
	r.POST("/api/v1/auth/login", authHandler.Login)
	r.POST("/api/v1/auth/refresh", authHandler.Refresh)
	r.POST("/api/v1/auth/logout", authHandler.Logout)

	logger.Log.Info("Auth Service listening on port 8081...")
	if err := r.Run(":8081"); err != nil {
		log.Fatalf("Auth Service failed: %v", err)
	}
}
```

**B. User Repository (`auth-service/internal/auth/repository/user_repo.go`)**
```go
package repository

import (
	"context"
	"database/sql"
	"errors"

	"github.com/bashocode/gowallet/microservices/auth-service/internal/auth/model"
)

type UserRepository interface {
	GetByEmail(ctx context.Context, email string) (*model.User, error)
	GetByID(ctx context.Context, id string) (*model.User, error)
}

type mysqlUserRepository struct {
	db *sql.DB
}

func NewMySQLUserRepository(db *sql.DB) UserRepository {
	return &mysqlUserRepository{db: db}
}

func (r *mysqlUserRepository) GetByEmail(ctx context.Context, email string) (*model.User, error) {
	query := `SELECT id, full_name, email, role, password_hash, avatar_url, is_verified, created_at, updated_at FROM users WHERE email = ? AND deleted_at IS NULL`
	u := &model.User{}

	err := r.db.QueryRowContext(ctx, query, email).Scan(
		&u.ID, &u.FullName, &u.Email, &u.Role, &u.PasswordHash, &u.AvatarURL, &u.IsVerified, &u.CreatedAt, &u.UpdatedAt,
	)
	if err != nil {
		if errors.Is(err, sql.ErrNoRows) {
			return nil, errors.New("user not found")
		}
		return nil, err
	}
	return u, nil
}

func (r *mysqlUserRepository) GetByID(ctx context.Context, id string) (*model.User, error) {
	query := `SELECT id, full_name, email, role, password_hash, avatar_url, is_verified, created_at, updated_at FROM users WHERE id = ? AND deleted_at IS NULL`
	u := &model.User{}

	err := r.db.QueryRowContext(ctx, query, id).Scan(
		&u.ID, &u.FullName, &u.Email, &u.Role, &u.PasswordHash, &u.AvatarURL, &u.IsVerified, &u.CreatedAt, &u.UpdatedAt,
	)
	if err != nil {
		if errors.Is(err, sql.ErrNoRows) {
			return nil, errors.New("user not found")
		}
		return nil, err
	}
	return u, nil
}
```

### Step 5: Dockerfile & Docker Compose
Create `auth-service/Dockerfile`. Note that `api-gateway` and `shared` are copied to builder context to satisfy `go.work` dependencies:

```dockerfile
# === Stage 1: Builder ===
FROM golang:1.26.4-alpine AS builder
WORKDIR /app

# Copy go.work and local module dependencies
COPY go.work ./
COPY shared/ ./shared/
COPY auth-service/ ./auth-service/
COPY api-gateway/ ./api-gateway/

WORKDIR /app/auth-service
RUN CGO_ENABLED=0 GOOS=linux GOARCH=amd64 go build -ldflags="-w -s" -o /service ./cmd/main.go

# === Stage 2: Production Image ===
FROM alpine:3.24.1
RUN apk --no-cache add ca-certificates tzdata

WORKDIR /app
COPY --from=builder /service .

EXPOSE 8081
CMD ["./service"]
```

### Step 6: Fix API Gateway Reverse Proxy
Since Go 1.20+, configuring `Rewrite` on a reverse proxy initialized with `httputil.NewSingleHostReverseProxy` raises an error due to both `Director` and `Rewrite` being set. We resolve this by instantiating `httputil.ReverseProxy` directly using the `Rewrite` field:

```go
	// Create Go's built-in reverse proxy with Rewrite only
	proxy := &httputil.ReverseProxy{
		Rewrite: func(r *httputil.ProxyRequest) {
			r.SetURL(url)
			r.Out.Header.Set("X-Forwarded-Host", r.In.Header.Get("Host"))

			// Inject & forward Request Correlation ID for distributed logging
			corID := r.In.Header.Get("X-Correlation-ID")
			if corID == "" {
				corID = uuid.New().String()
			}
			r.Out.Header.Set("X-Correlation-ID", corID)
		},
	}
```
